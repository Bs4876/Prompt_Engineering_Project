# Prompt Engineering – CLI Agent

## תיאור הפרויקט
בפרויקט זה פיתחתי Agent שמקבל הוראה בשפה טבעית וממיר אותה לפקודת CLI של Windows.

הדגש בפרויקט היה על תהליך של Prompt Engineering איטרטיבי:
בדיקה → ניתוח → שיפור הפרומפט → בדיקה מחדש.

האפליקציה בנויה עם Python ו-Gradio ומבצעת קריאה ל-LLM שמחזיר את פקודת ה-cmd המתאימה.

## איטרציות של הפרומפט

במהלך העבודה ביצעתי מספר איטרציות לשיפור הפרומפט:

- Level 1 – גרסה בסיסית שממירה הוראה לפקודת CLI.
- Level 2 – הוספת מנגנון סיווג בטיחות (SAFE / WARNING / BLOCKED).
- Level 3 – תמיכה במקרים מורכבים יותר כמו chaining ו-pipes.
- Level 4 – שיפור אופטימיזציה, בטיחות ודיוק הפלט.

---

## כשלים מעניינים ומה למדתי

### 1. המודל מוסיף הסברים
לפעמים המודל החזיר טקסט נוסף לפני הפקודה.

מה למדתי:
צריך להגדיר בצורה מאוד מפורשת בפרומפט שהפלט חייב להיות שורת פקודה אחת בלבד ללא הסברים.

### 2. הוראות לא ברורות
כאשר ההוראה לא הייתה ספציפית (לדוגמה: "delete file"), המודל ניסה לנחש את הפקודה.

מה למדתי:
צריך להגדיר שהמודל יחזיר הודעת UNCLEAR כאשר אין מספיק מידע.

### 3. פעולות מסוכנות
בגרסאות הראשונות המודל החזיר פקודות מחיקה ללא אזהרה.

מה למדתי:
יש צורך במנגנון סיווג בטיחות שמבדיל בין פקודות SAFE, WARNING ו-BLOCKED.

### 4. משימות מורכבות
בהתחלה המודל התקשה במשימות שדורשות יותר מפקודה אחת.

מה למדתי:
הוספת תמיכה ב-pipes ו-command chaining שיפרה את היכולת להתמודד עם משימות מורכבות.
